Compression & Expansion Rules
  • 1
    Remove leading zeros from each 16-bit group. 0000→0, 0db8→db8, 0001→1. At least one digit must remain per group.
  • 2
    Replace the longest run of consecutive all-zero groups with ::. The run must be 2 or more consecutive 0000 groups. :: can only appear once in an address.
  • 3
    If two zero-runs are equal length, replace the first one (RFC 5952). A single isolated zero group must not be shortened to ::.
  • 4
    To expand: pad every group to 4 hex digits; replace :: with enough 0000 groups so that the total reaches 8.
    2001:db8::1 → 2001:0db8:0000:0000:0000:0000:0000:0001
    fe80::a1b2:1 → fe80:0000:0000:0000:0000:0000:a1b2:0001
