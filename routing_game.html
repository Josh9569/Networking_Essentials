Administrative Distance Reference

When two different routing sources learn a route to the same network, the router installs the one with the lowest administrative distance (AD). Lower always wins — it's a measure of trustworthiness, not speed.

Route Table Anatomy

Every dynamic route line follows the same shape. Static/connected lines drop the metric or next-hop where they don't apply.

O   192.168.20.0/24 [110/2] via 10.0.0.2, GigabitEthernet0/1
  • 1O — the source code letter (see AD Reference). Tells you which protocol installed this route.
  • 2192.168.20.0/24 — the destination network and prefix length this entry covers.
  • 3[110/2] — bracketed pair of [Administrative Distance / Metric]. AD ranks the source; metric ranks the path within that source.
  • 4via 10.0.0.2 — the next-hop IP address the router forwards matching packets to. Directly connected routes have no next-hop, only an interface.
  • 5GigabitEthernet0/1 — the exit interface used to reach that next-hop.
Longest Prefix Match Rule
  • 1A router compares a packet's destination IP against every route in the table and keeps every entry whose network/mask contains that address.
  • 2Among all matching entries, the router always uses the one with the longest prefix (the highest /N, i.e. the most specific, smallest network) — regardless of AD or metric.
  • 3Administrative distance only breaks a tie when two different sources advertise the exact same prefix — it never overrides a longer match.
  • 4If nothing else matches, the default route 0.0.0.0/0 (if present) catches everything. If even that's missing, the packet is dropped.
