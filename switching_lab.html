VLAN / Trunk Reference
  • 1
    An access port belongs to exactly one VLAN and connects end devices (PCs). Frames on it are untagged — the switch tracks the VLAN internally.
    switchport mode access · switchport access vlan 10
  • 2
    A trunk port carries multiple VLANs between switches using 802.1Q tags. Both ends of the link must be trunks or traffic will not pass correctly.
    switchport mode trunk
  • 3
    Two PCs can only communicate at Layer 2 if they are in the same VLAN and every switch-to-switch link on the path carries that VLAN.
  • 4
    EtherChannel bundles multiple physical links into one logical link (more bandwidth, no STP blocking between them). Every member port on a switch must share the same channel-group number and identical configuration.
    channel-group 1 mode on
STP Rules
  • 1
    Root bridge election: the switch with the lowest bridge ID wins. Bridge ID = priority + MAC address. Compare priority first; if tied, the lowest MAC wins.
  • 2
    Every non-root switch picks one root port — the port with the lowest total path cost to the root (FastEthernet link = cost 19).
  • 3
    Each link segment gets one designated port — the end whose switch has the lower path cost to the root; ties are broken by lower bridge ID. All root-bridge ports are designated.
  • 4
    Any port that is neither a root port nor a designated port becomes blocking — this breaks the loop. In a triangle of switches exactly one port blocks: on the redundant link, the end belonging to the switch with the higher bridge ID.
  • 5
    Priorities can also be set with keywords: root primary configures priority 24576 and root secondary configures 28672, both lower (better) than the default 32768. Lower always wins the election.
    spanning-tree vlan 1 root primary
Command Reference

The CLI accepts Cisco IOS syntax with standard abbreviations (e.g. conf t, int fa0/1, sw mo ac). Commands work at any prompt level for convenience. Press Tab to auto-complete keywords and use the ↑ / ↓ arrow keys for command history.

configure terminalEnter config mode (conf t)
vlan <n>Create VLAN n and enter config-vlan mode
name <text>Name the VLAN (from config-vlan mode)
interface fa0/<n>Select a port (int fa0/1)
switchport mode accessMake the port an access port (sw mo ac)
switchport access vlan <n>Assign VLAN (sw ac vl 10)
switchport mode trunkMake the port a trunk (sw mo tr)
channel-group <n> mode onAdd port to EtherChannel n
no switchport access vlanReturn port to VLAN 1
no channel-groupRemove from EtherChannel
show vlan briefShow VLAN → port table
show runShow this switch's port config
exit / endGo up one level / back to exec
